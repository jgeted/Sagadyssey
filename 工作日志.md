# 工作日志

## 2026-06-14 — 项目启动与需求梳理

**做了什么**

- Claude 通读并分析了 AncientWarfare2 模组的全部核心源码（600+ Java 文件，5 个模块）
- 输出了完整的技术分析：架构、网络系统、注册表加载器、所有权系统、研究系统、数据修复等
- 阅读了 jgeted 的"一个想法.docx"，逐轮提问，完成了项目需求梳理
- 输出了《项目梳理.md》——包含项目骨架、四模块定义、Core v0.1.0 边界、待思考问题、下一步建议
- 确认了开发顺序：先做微型模组走通流程 → Core → NPC / Structure / Vehicle → Automation 以后再说

**关键决策**

- 不做 AW2 级的重型轮子（37 种数据包、.aws 解析器、派系声望数值等）
- NPC：全新实体类，6-8 种，村庄+营地自然生成，付费招募，GUI+物品切换职业，简单敌友关系
- Structure：简单预设结构，建造工具用比 AW2 更轻量的方案
- Vehicle：增强原版坐骑（马、驴、狗、船、箱子船），改进寻路 AI
- Core v0.1.0 真正必须的：网络框架 + 配置管理 + 研究系统（技能树模式）

## 2026-06-15 — 全部决策敲定

**做了什么**

- 定下模组名称：**Sagadyssey**（史诗远行），GitHub：jgeted/Sagadyssey，MOD_ID：`sagadyssey`
- 对比 NeoForge vs Fabric，结合项目需求选定 NeoForge

- 逐项敲定了研究点数、NPC 人口控制、Structure 建造工具方案
- 更新了《项目梳理.md》和持久记忆

**最终决策总表**

| 决策项 | 决定 | 备注 |
|---|---|---|
| 模组名称 | **Sagadyssey**（史诗远行） | MOD_ID：`sagadyssey`，GitHub：jgeted/Sagadyssey |
| Minecraft 版本 | **1.21.1**（NeoForge 稳定版） | 26.1 稳定后迁移 |

| Structure 建造工具 | 先做 B 方案 | 手持建筑图纸 + 半透明预览 + 确认放置。C 方案（扫描器）等模组基本成型后再考虑 |
| 开源协议 | 发布前再定 | 不影响开发 |
