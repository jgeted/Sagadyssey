## 二、微型模组（第一步）

不是 Core——比 Core 更小。目标：走通 1.21.1 NeoForge 开发的完整流程。

**内容：** 加一个方块，右键打开 GUI，GUI 里有一个按钮，点了之后在聊天栏输出一句话。

**涉及的技术：** 环境搭建、注册方块、注册 GUI、网络通信、客户端服务端同步。

**预计：** AI 辅助约 1-2 周（每周 8 小时）。做完再评估 Core 的开发计划。

## 四、NPC 模块

### 基础设定

- **6-8 种**全新实体类，不基于原版生物（工人、战士、弓箭手等）
- **派系系统：** v0.1 只做简单敌友关系，以后可能扩展
- **操控方式：** 物品控制
- **战斗 AI：** 尽量复用原版 AI Goal，不够用才自己写

### 获取与招募

- **生成位置：** 已有村庄 + 野外独立生成点
- **招募方式：** 右键付费（绿宝石/物品），以后可能加其他方式

### 职业切换

两种方式并行：
- GUI 界面选择职业
- 给不同物品自动转职：给剑 → 战士、给锄 → 农民、给弓 → 弓箭手……

### 工作能力

- 采集资源、搬运物品、自动生产
- 各职业之间可以互相转换

### 人口控制——中世纪题材双方案

**小型营地（5 种）**

| 类型 | NPC 数 | 结构 |
|---|---|---|
| 盗匪营地 | 3-5 | 帐篷、篝火、马车、粗木栅栏 |
| 猎人小屋 | 1-2 | 木屋、晾肉架、猎犬 |
| 行商歇脚点 | 2-3 | 篷车、篝火、驮马 |
| 瞭望哨 | 2-3 | 木塔、篝火 |
| 隐士居所 | 1 | 石砌小屋、药草园 |

人口逻辑：结构生成时一次性刷满，不自动补人。NPC 被招募或死亡后营地逐渐空置，一段时间后消失。

**大型定居点（4 种）**

| 类型 | NPC 数 | 结构 |
|---|---|---|
| 围墙村庄 | 8-15 | 民居、铁匠铺、小教堂、市场、水井 |
| 边境要塞 | 6-10 | 石墙、营房、军械库、瞭望塔 |
| 修道院 | 4-8 | 教堂、藏经阁、药草园、僧舍 |
| 领主庄园 | 5-8 | 主宅、马厩、佣人房、磨坊 |

人口逻辑：床位绑定上限，死亡后冷却补位（每 5 分钟补 1 个），招募后床位腾出自动补。

**死亡处理：** NPC 死亡即消失，不做特殊处理。

**开发监控命令：** `/npc pop` 查看人口 / `/npc pop nearby` 附近 / `/npc pop camp` 营地 / `/npc pop settlement` 定居点

## 五、Structure 模块

### 世界生成

简单预设结构，固定生成规则。以后可能扩展为完整配置系统。

### 建造工具——B 方案

手持"建筑图纸"物品 → 右键打开 GUI 选结构 → 世界中出现半透明预览 → 左键确认放置、右键取消。

底层利用 1.21+ 原版结构放置机制，不造 .aws 解析器。C 方案（扫描器 + 图纸化）等模组基本成型后再考虑。

## 七、明确不做或推迟的

- AW2 级的重型网络框架（37 数据包 + 63 GUI ID）
- .aws 模板格式和完整 StructureBuilder

- C 方案扫描器
- 难度倍率
- 音效和粒子效果（目前暂不列入，以后需要再加）

## 八、潜在风险（开发中留意）

- **Mixin：** 增强坐骑和 AI 的核心手段，有自己的语法和调试方式，两个模组 Mixin 同一个方法会冲突
- **GUI 复杂度：** 技能树、职业切换 GUI 涉及 Screen 系统、坐标计算、按钮事件、滚动区域
- **寻路改进：** 原版 PathNavigate/PathFinder 改造困难，很多大型模组选择绕过。需要定"改到什么程度算够了"的上限
- **世界生成：** 即使简单预设也需选择——原版 datapack 结构系统（省力但有限制）或自己接管（灵活但代码多）
- **多人测试：** NPC 系统、派系敌友在单人里测不出问题，尽早双人测试
