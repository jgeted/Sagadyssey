三、架构
1. Core（核心模块）— a（完善）
所有子模块的强制依赖，提供了整个模组的基础框架：
•	科技系统（ResearchTracker）：玩家需要通过点亮科技树获得对应的兵种，武器。
•	手册系统：游戏内指南手册。
•	网络系统（NetworkHandler）：自定义网络通道 "AWCORE"，定义了 37 种数据包类型和 63 种 GUI ID。
•	JSON 注册表加载器（RegistryLoader）：从 JAR 资源和配置覆盖文件夹中加载 JSON 注册表数据，支持依赖排序、mod 条件过滤。
•	所有权系统（Owner）：基于 UUID + 名称的不可变所有权对象，支持队伍/好友检查。
•	配置管理（AWCoreStatics / ModConfiguration）：分层的配置系统，支持每个物品的研究/可合成性配置。
•	兼容层（CompatLoader / ICompat）：插件化的兼容接口，已实现 FTB 队伍兼容（FTBCompat）、JEI 配方集成。
•	数据修复（AWDataFixes）：Forge DataFixer 集成，包括 TileId 修复、TileOwner 修复、VehicleOwner 修复、ResearchNote 修复。
2. NPC（NPC 模块）— ancientwarfarenpc
添加完整的派系 NPC 系统：
•	NPC 实体（NpcBase，1459 行）：继承 EntityCreature，包含： 
o	AI 任务位域系统
o	等级/经验值系统（NpcLevelingStats）
o	皮肤系统（NpcSkinSettings）
o	自定义战斗系统（复制 vanilla 攻击逻辑 + 盾牌格挡 AI + 方块拆除/放置）
o	玩家可以命令NPC进行对应的动作（移动到哪里，攻击哪里）
o	护甲计算
o	睡眠/归位系统
o	重新打包为物品（repack）
o	载具骑乘集成
•	派系系统（FactionTracker / FactionData / TeamData）：管理派系声望值，支持队伍和个人两种声望。取最低值判定敌对关系。
•	NPC 类型：工人（工作 NPC）、战斗 NPC（弓箭手、士兵、骑兵、后勤兵）。
•	交易系统：派系交易和玩家拥有的 NPC 交易。
•	市政厅（ContainerTownHall）：管理 NPC 和派系。
•	Minecraft 命令：/faction（派系管理）、/teams（队伍管理）、/debugAI（AI 调试）。
•	兼容层：EpicSiege、TwilightForest、EBWizardry（使用 Lambda 延迟加载避免 ClassNotFoundException）。
3. Structure（结构模块）— ancientwarfarestructure
添加结构模板和世界生成系统：
•	结构模板系统（StructureTemplateManager / StructureBuilder）： 
o	模板文件使用 .aws 格式（自定义文本格式，含 layers/blocks 定义）
o	StructureBuilder 支持即时建造（instantConstruction），按优先级分层建造
o	支持旋转（东南西北四个方向）和生物群系方块替换（沙漠→砂岩、针叶林→云杉等）
o	支持 biome 替换和特殊方块处理（如铁轨需要 set 两次）
o	实体规则支持（在结构中放置 NPC、怪物等）
•	世界生成（WorldStructureGenerator / WorldTownGenerator）： 
o	可配置的世界结构生成
o	城镇生成器（Town Generator）
o	WorldGenTickHandler：分帧处理世界生成任务，避免卡顿
•	模板数据修复系统（DataFixManager）：10 个修复器处理版本升级（方块 Meta→BlockState、实体装备、实体位置→NBT、派系扩展、规则名合并、JSON 简化等）。
•	建筑方块和装饰方块（40+ 方块）：椅子、宝座、桌子、笼子、火盆、图腾、奖杯、讲台、祭坛、门、头颅、战利品箱、木桩、雕像、声音方块、结构扫描器等。
•	实体：EntityGate（大门实体）、EntitySeat（座椅实体）。
•	高级生成器（ContainerSpawnerAdvanced）：可自定义的实体生成器。
•	扫描器（ContainerStructureScanner）：扫描世界区域保存为模板。
•	蓝图站（ContainerDraftingStation）：编辑和管理模板。
•	MinifyStructuresTask：自定义 Gradle 任务，压缩 .aws 模板文件（合并相同的相邻方块/行/层，减小文件体积）。
4. Vehicle（载具模块）— ancientwarfarevehicle(完善)
添加攻城武器和运输载具：
•	载具实体（VehicleBase，1095 行）：可骑乘的实体，包含： 
o	炮塔系统：本地俯仰/偏航和插值动画
o	移动系统：前进/后退/侧移速度，受材料倍率影响
o	弹药系统：VehicleAmmoHelper 管理弹药类型和数量
o	射击系统：VehicleFiringHelper 计算弹道轨迹
o	升级系统：VehicleUpgradeHelper 管理升级效果
o	装甲系统：Minecraft 自带的盔甲
•	载具类型（IVehicleType 的 20+ 种实现）： 
o	马、驴、狗等Minecraft自带生物
o	船，箱子船
•	弹药类型：箭矢等
•	网络数据包：瞄准更新、弹药选择/更新、射击更新、升级更新、载具输入/移动、打包命令

四、关键技术系统详解（重写）
网络架构（NetworkHandler）
•	通道名："AWCORE"
•	37 种数据包类型，63 种 GUI ID
•	统一的消息路由：sendToServer、sendToPlayer、sendToAllPlayers、sendToAllTracking
•	服务器端容器创建和客户端 GUI 创建均通过反射实现模块化解耦
•	@GuiSide 和 @NetworkSide 注解支持
JSON 注册表加载器（RegistryLoader）
•	使用 Gson 解析
•	支持从 JAR 资源和配置覆盖文件夹两个来源加载
•	文件依赖排序（load_after）
•	条件加载（disabled 标志 + mod 依赖检查）
•	去重（基于 ResourceLocation 的映射）
•	各模块注册的解析器包括：研究数据（Research）、作物（Crop/Soil/Tillable）、果树（Fruit）、树木（Plantable/Soil/Scanner）、派系（Faction）、目标列表（TargetList）、NPC 默认值、派系交易列表、实体生成 NBT、生物群系组、结构方块注册、领地设置
所有权与队伍系统
•	Owner：不可变类（UUID + 名称），有 EMPTY 哨兵值
•	TeamViewerRegistry：抽象队伍/好友检查接口，支持 FTB 队伍兼容
•	权限检查链：所有者 → 队伍 → 好友
研究系统
•	客户端：ResearchTracker.clientData
•	服务器：通过 AWGameData 持久化到世界存档
•	研究队列管理和进度追踪
•	与合成系统深度集成：配方匹配时检查研究完成度
数据修复管线
•	Core 模块：TileId → TileOwner → VehicleOwner → ResearchNote
•	Structure 模块：FactionExpansion → JsonSimplification → BlockMetaToBlockState → EntityPositionToNBT → RuleNameConsolidation → EntityRuleName → EntityEquipment → TileLoot → LootSettingsPotionRegName → WoodenCoffin
